# Brief AI — MVP

Интеллектуальный ассистент для генерации технических заданий.  
AI заменяет роль Product Owner и бизнес-аналитика на этапе создания ТЗ.

## Быстрый старт

```bash
# 1. Установка зависимостей
npm install

# 2. Настройка окружения
cp .env.example .env
# Указать: DATABASE_URL, ANTHROPIC_API_KEY, REDIS_URL

# 3. Миграция БД
npm run db:migrate

# 4. Наполнение FeatureCatalog
npm run db:seed

# 5. Запуск
npm run dev
```

## Ключевые решения

| Решение | Почему |
|---------|--------|
| Отдельный промпт на шаг | Контроль роли AI, размера контекста, тестируемость |
| context_summary + structured_facts | Summary для контекста, facts для критичных данных (не теряются) |
| FeatureCatalog в БД, не в промпте | Масштабируемость, обновление без редеплоя, основа для шаблонов |
| Wizard Engine на JSON-конфиге | Новые типы брифов без переписывания кода |
| Override warnings | Пользователь не застревает, но разработчик видит пометки |

## Команда

1. **Техлид** (Senior) — архитектура, БД, API, деплой
2. **Frontend** (Middle+) — wizard UI, онбординг, адаптив
3. **AI/Prompt-инженер** (Senior) — промпты, eval, качество
4. **PO** (50%) — FeatureCatalog, приёмка, метрики
5. **UI/UX** (50%) — макеты, дизайн-система
6. **Backend голос** (фаза 2) — WebSocket, STT, TTS
